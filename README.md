# Интеллектуальная платформа подготовки к ОГЭ и ЕГЭ по математике

Учебный проект представляет веб-платформу для подготовки к ОГЭ и ЕГЭ по математике. Система фиксирует процесс выполнения задания, хранит историю попыток, принимает фото или PDF решения, подключает ИИ-агентов для анализа работы и формирует аналитику подготовки.

## Цель проекта

Создать прозрачную среду подготовки к экзаменам, в которой ученик решает задания в контролируемом сценарии, преподаватель видит ход работы, а родитель получает понятный отчет о динамике подготовки.

## Основные возможности

- два курса: подготовка к ОГЭ по математике и подготовка к ЕГЭ по математике;
- структура курса по экзаменационным номерам, темам и заданиям;
- отображение заданий как изображений из импортированных банков;
- таймер выполнения задания;
- загрузка одной или нескольких страниц решения в формате JPG, PNG или PDF;
- сохранение истории попыток без удаления предыдущих результатов;
- анализ решения через ИИ-агента;
- учебный чат с ИИ-тьютором;
- панель преподавателя с просмотром работ, файлов решений, истории попыток и комментариев;
- родительский отчет;
- аналитика подготовки и прогноз результата;
- административное добавление эталонных решений и корректировка заданий.

## Архитектура

```text
Пользовательский интерфейс
        |
        v
FastAPI backend
        |
        +-- SQLite / SQLAlchemy
        +-- сервис заданий и попыток
        +-- сервис аналитики и прогноза
        +-- AIClient для подключения внешних API
        |
        +-- TopicGuardAgent
        +-- TutorChatAgent
        +-- TextRecognitionAgent
        +-- SolutionReviewAgent
        +-- TaskGeneratorAgent
```

`AIClient` не является отдельным агентом. Это инфраструктурный слой, через который предметные агенты обращаются к внешним провайдерам.

## Роли пользователей

- Ученик: выбирает курс, открывает задания, запускает таймер, отправляет решение и получает комментарий.
- Преподаватель: просматривает отправленные работы, файлы решений, историю попыток, чат ученика и выставляет итоговый статус.
- Родитель: смотрит понятный отчет о прогрессе, рисках и прогнозе результата.
- Администратор: управляет заданиями, ответами, эталонными решениями и тестовыми пользователями.

## ИИ-агенты

- `TopicGuardAgent` контролирует учебную тематику обращения ученика.
- `TutorChatAgent` ведет учебный диалог и помогает разобраться в теме без прямой подмены самостоятельного решения.
- `TextRecognitionAgent` распознает текст из фото или PDF решения при подключенном OCR-провайдере.
- `SolutionReviewAgent` анализирует решение ученика, критерии и эталонное решение.
- `TaskGeneratorAgent` формирует похожие задания для закрепления навыка.

Дополнительные сервисы:

- `ChatMemoryService` хранит историю диалога и краткое содержание длинного общения.
- `ExamForecastService` рассчитывает показатели подготовки и прогноз результата.

## Настройка `.env`

1. Скопируйте файл `.env.example` в `.env`.
2. Заполните ключи тех провайдеров, которые будут использоваться.
3. Настоящий `.env` не должен публиковаться в репозитории.

Минимальный вариант для локального запуска без внешних API:

```env
DATABASE_URL=sqlite:///./exam_preparation.db
AI_MOCK_MODE=true
```

Пример настройки OCR через Yandex Cloud:

```env
YANDEX_API_KEY=your_yandex_api_key_here
YANDEX_FOLDER_ID=your_yandex_folder_id_here
TEXT_RECOGNITION_PROVIDER=yandex_ocr
TEXT_RECOGNITION_MODEL=page
```

Пример настройки OpenAI-совместимого провайдера для ИИ-агентов:

```env
POLZA_API_KEY=your_polza_ai_key_here

TOPIC_GUARD_PROVIDER=openai
TOPIC_GUARD_API_KEY=your_polza_ai_key_here
TOPIC_GUARD_BASE_URL=https://polza.ai/api/v1
TOPIC_GUARD_MODEL=openai/gpt-4.1-nano

TUTOR_CHAT_PROVIDER=openai
TUTOR_CHAT_API_KEY=your_polza_ai_key_here
TUTOR_CHAT_BASE_URL=https://polza.ai/api/v1
TUTOR_CHAT_MODEL=openai/gpt-4.1-nano

SOLUTION_REVIEW_PROVIDER=openai
SOLUTION_REVIEW_API_KEY=your_polza_ai_key_here
SOLUTION_REVIEW_BASE_URL=https://polza.ai/api/v1
SOLUTION_REVIEW_MODEL=openai/gpt-4.1-mini

TASK_GENERATOR_PROVIDER=openai
TASK_GENERATOR_API_KEY=your_polza_ai_key_here
TASK_GENERATOR_BASE_URL=https://polza.ai/api/v1
TASK_GENERATOR_MODEL=openai/gpt-4.1-nano
```

Состояние подключений можно проверить по адресу:

```text
http://127.0.0.1:8000/api/ai/health
```

Ключи и секретные значения в ответе этого маршрута не выводятся.

## База данных и стартовые данные

При первом запуске создается SQLite-база `exam_preparation.db`. Если база уже существует, приложение использует текущие данные.

Стартовые данные включают:

- курсы ОГЭ и ЕГЭ по математике;
- экзаменационные разделы и темы;
- задания из импортированных банков;
- тестовые учетные записи для ролей ученика, преподавателя, родителя и администратора;
- недельный план подготовки.

Изображения условий заданий хранятся в `static/imported_tasks/` и нужны для корректного отображения заданий.

## Сценарий проверки

1. Открыть `http://127.0.0.1:8000/`.
2. Выбрать ученика.
3. Открыть курс ОГЭ или ЕГЭ по математике.
4. Перейти в модуль и открыть задание.
5. Нажать «Начать попытку».
6. Убедиться, что таймер начал отсчет.
7. Загрузить одну или несколько страниц решения.
8. Нажать «Отправить на проверку».
9. Посмотреть результат проверки и комментарий ИИ.
10. Открыть ИИ-тьютор и задать вопрос по заданию.
11. Перейти в панель преподавателя.
12. Открыть работу ученика, просмотреть файл решения, историю попыток и комментарий ИИ.
13. Добавить комментарий преподавателя и итоговый статус.
14. Открыть родительский отчет и аналитику подготовки.

## Импорт банков заданий

В проекте предусмотрен технический скрипт для импорта PDF-банков заданий:

```bash
python scripts/import_math_bank.py --file "path/to/oge_bank.pdf" --exam OGE
python scripts/import_math_bank.py --file "path/to/ege_profile_bank.pdf" --exam EGE_PROFILE
```

Для работы `pdf2image` требуется установленный Poppler. Если он не найден автоматически, укажите путь:

```bash
python scripts/import_math_bank.py --file "path/to/oge_bank.pdf" --exam OGE --poppler-path "path/to/poppler/bin"
```

Также можно задать переменную окружения:

```env
POPPLER_PATH=path/to/poppler/bin
```

## Основные маршруты

- `GET /` — главная страница;
- `GET /courses` — список курсов;
- `GET /courses/{course_id}` — страница курса;
- `GET /courses/{course_id}/sections/{section_id}` — страница модуля;
- `GET /tasks/{task_id}` — страница задания;
- `POST /tasks/{task_id}/start` — начало попытки;
- `POST /attempts/{attempt_id}/commit` — отправка решения;
- `GET /teacher/attempts` — панель преподавателя;
- `POST /ai/chat` — учебный чат;
- `GET /users/{user_id}/analytics` — аналитика подготовки;
- `GET /users/{user_id}/forecast` — прогноз результата;
- `GET /users/{user_id}/parent-report` — отчет для родителя.

## Безопасность конфигурации

- `.env` добавлен в `.gitignore` и не должен публиковаться.
- В `.env.example` находятся только шаблонные значения.
- Загруженные учеником решения хранятся в `uploads/`; папка исключена из публикации.
- В пользовательском интерфейсе не отображаются ключи внешних API.
- Логи и временные файлы исключены из репозитория.
